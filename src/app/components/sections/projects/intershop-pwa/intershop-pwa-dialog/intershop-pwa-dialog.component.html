Formly Integration

The Problem:
We realized that our complex network of custom form components, including validation and styling, was becoming increasingly hard to maintain and customize - both for ourselves and for our clients. We needed a flexible, easily customizable way to write forms.

The Solution:
I researched the Formly library which allows for programmatic definition of forms via typescript. After a usability test, I developed a solution - based on Formly - that enables clients to easily define custom registration forms and country-specific address templates. I also held a series of trainings for my team to explain Formly and its implementation. We've been converting every single form since.

Localization Update

The Problem:
A common criticism with the Intershop PWA was the fact that changing even a single translation required a change to a set of local json files. This meant that every translation change required a new deployment, which created a lot of work for our clients as well as operations team.

The Solution:
I researched multiple solutions for enabling dynamically loaded translations. After several implementation tests, we decided on loading them via the Intershop REST API. I created a custom ngx-translate loader to communicate with the API and use the local files as a fallback. In addition, I wrote a simple script which converts json files to the xliff format that is used by Intershop's translation management backend. We received very positive feedback for the flexible solution.

TSLint to ESLint migration

The Problem:
Angular 12 deprecated the use of TSLint as their default linting solution. To stay current, we have to migrate to their new solution, general-purpose ESLint. A big project, the Intershop PWA comes with a large amount of custom lint rules. The entire configuration as well as all custom rules have to be migrated or rewritten and the source code adapted.

The Solution:
I researched migration tools to aid with an automated migration to an ESLint configuration. These did part of the job but not every TSLint rule has an ESLint equivalent. In addition, I studied our configuration in depth and found equivalent rules or wrote new rules where appropriate. I went through the @typescript-eslint source code and am using their experimental utils library to rewrite all custom rules and fixers in an organized, maintanable manner.
